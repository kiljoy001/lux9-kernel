## Implementation Analysis

### High-Priority Algorithms for OpenSSL Replacement

#### Stream Ciphers (crypto_stream)
- **chacha20**: Modern, fast, secure stream cipher
- **salsa20**: Predecessor to ChaCha20, well-analyzed
- **aes128ctr**, **aes256ctr**: AES in counter mode

#### Hash Functions (crypto_hash)
- **sha256**, **sha512**: SHA-2 family (NIST standard)
- **sha3_256**, **sha3_512**: SHA-3 family (NIST standard)
- **blake2b**, **blake2s**: Fast, secure hash functions

#### KEMs (crypto_kem)
- **kyber512**, **kyber768**, **kyber1024**: NIST ML-KEM
- **frodokem**: Conservative lattice-based KEM
- **ntru**: NTRU-based post-quantum KEM

#### Signatures (crypto_sign)
- **dilithium2**, **dilithium3**, **dilithium5**: NIST ML-DSA
- **sphincsshake256**: NIST SLH-DSA (SPHINCS+)
- **ed25519**: Fast elliptic curve signatures

#### AEAD (crypto_aead)
- **chacha20poly1305**: ChaCha20-Poly1305 (RFC 7539)
- **aes256gcm**: AES-256-GCM (NIST standard)
- **aegis128**, **aegis256**: High-performance AEAD

### Reference Implementation Strategy

1. **Extract source code** for priority algorithms
2. **Study optimizations** in different implementations
3. **Adapt for kernel space** with formal verification
4. **Validate with SUPERCOP test vectors**

### Performance Baselines

SUPERCOP provides performance measurements on various architectures:
- Use as baseline for our kernel implementation
- Target 2-5x improvement with kernel-space optimizations
- Maintain or exceed security properties
